# 前端整体重新设计

## 1. 背景

一期前端以最小可运行为目标，所有页面共享"标题 + 表格 + 弹窗"模式，存在以下问题：

- 仪表盘只有数字指标和入口按钮，缺少有价值信息
- 完成一个流程需要在多个页面之间切换
- 创建操作全部使用弹窗，打断上下文
- 所有页面结构单调重复，信息密度低
- 视觉风格偏暖色调，不够现代精致

本设计对前端进行整体重新设计，采用 Linear / Notion 风格，以工作流为导向重构所有页面。

## 2. 目标与非目标

### 2.1 目标

- 建立全新的 Linear 风格设计系统（冷灰调、蓝色主色、紧凑圆角）
- 重写 AppShell 布局（紧凑顶栏）
- 业务页面统一采用列表+详情面板双栏交互
- 仪表盘改为角色工作流看板
- 弹窗操作改为面板内联表单
- 新增通用组件体系（MasterDetail、ItemCard、DetailPanel 等）
- 所有 10 个页面全部重写
- 集成处理器协议带来的前端变化
- 响应式适配

### 2.2 非目标

- 实时推送 / WebSocket 通知
- 全局搜索和快捷键系统
- 暗色主题切换
- 国际化 (i18n)
- 复杂页面过渡动画

## 3. 技术栈

- Vue 3 + TypeScript + Vite（不变）
- Element Plus（保留，做深度样式定制）
- 引入 Inter 字体（Google Fonts）
- Vanilla CSS 变量体系

### 4.1 Design Tokens

```css
  --bg-surface:     #ffffff;

  /* 边框 */

  --text-tertiary:  #9ca3af;

  /* 主色 */

```

### 4.2 间距体系

基于 4px 倍数：4, 8, 12, 16, 20, 24, 32, 48。

### 4.3 Element Plus 深度定制

- 按钮：使用 `--accent` 主色覆盖默认蓝
- 表格：去掉斑马纹默认色，使用 `--bg-hover`
- 对话框：统一 `--radius-lg` 圆角
- 表单项：缩小默认间距

## 5. 整体布局

### 5.1 AppShell

```
┌─────────────────────────────────────────────────────┐
│  Logo   导航标签页      [角色徽章] [用户名] [退出]     │  48px 高度
├─────────────────────────────────────────────────────┤
│                                                     │
│                    内容区                             │
│                                                     │
└─────────────────────────────────────────────────────┘
```

- 顶栏高度 48px，白色背景，底部 1px 分隔线
- Logo 区域精简为文字标题
- 导航标签页紧凑排列，选中态用底部 2px 蓝色指示线
- 用户区域：角色徽章 + 用户名 + 退出按钮

### 5.2 业务页面双栏布局

```
┌─ 列表区（40%）────────┐ ┌─ 详情面板（60%）──────────┐
│ [搜索] [筛选] [+新建] │ │                          │
│                      │ │  标题 + 状态徽章           │
│  卡片列表             │ │  ──────────               │
│  (可滚动)             │ │  基本信息区                │
│                      │ │  操作区                    │
│                      │ │  关联数据区                 │
│                      │ │  (可滚动)                  │
└──────────────────────┘ └──────────────────────────┘
```

- 两栏之间用 1px 竖线分隔
- 列表区和详情面板各自独立滚动
- 无选中时详情面板显示 EmptyState 组件
- 移动端退化为单栏，点击列表项全屏展开详情

## 6. 仪表盘设计

### 6.1 核心理念

每个角色看到的是自己"下一步该做什么"的工作流看板，而非数字指标。

### 6.2 Provider 视图

三列看板：

- **待处理**：待审批的需求列表
- **进行中**：草稿状态的目录
- **近期完成**：已审批的需求、已发布的目录

底部附加活动时间线区域。

### 6.3 Aggregator 视图

三列看板：

- **待处理**：待审批的需求 + 可创建任务的已获批需求
- **处理中**：running 状态的任务（显示处理器名称和进度）
- **近期完成**：已完成的任务及产出信息

### 6.4 Admin 视图

三列看板：

- **待处理**：待审核的注册申请
- **系统状态**：在线处理器数量、活跃用户数、本周任务数
- **近期活动**：最近审计日志摘要

### 6.5 Consumer 视图

简洁的交付文件列表 + 下载操作，无需看板。

### 6.6 看板卡片设计

每张卡片显示：图标 + 标题摘要 + 次要信息 + 时间。可点击跳转到对应详情。

## 7. 业务页面详细设计

### 7.1 目录页

**列表区**：

- 搜索框 + 状态筛选下拉
- 卡片列表：每张卡片显示名称、版本、状态徽章（草稿/已发布/归档）、文件数量
- Provider 看自己的目录，Aggregator 看所有已发布目录

**详情面板**：

- 头部：目录名称 + 版本 + 状态徽章 + 操作按钮（发布/归档）
- 基本信息：数据类型、粒度、敏感级别、规模描述、字段说明、上传方式
- 文件列表区（FileList 组件）：显示目录下所有文件，支持追加上传和删除
- 关联需求区：展示该目录的所有需求，Provider 可快捷审批

**新建操作**：

- 点击 "+ 新建" 后详情面板变为内联创建表单
- 表单包含所有目录元数据字段
- 表单底部内嵌文件上传区域，支持多文件
- 提交后自动选中新创建的目录

### 7.2 需求页

**列表区**：

- 按状态分组展示：待审批 / 已获批 / 处理中 / 已交付 / 已驳回
- 每组标题显示数量
- 卡片：需求标题 + 目录名称 + 状态徽章

**详情面板**：

- 头部：需求标题 + 状态徽章
- 基本信息：目的、交付计划、对应目录摘要
- 状态流转时间线（申请 → 审批 → 数据就绪 → 处理中 → 已交付）
- 操作按钮区：
  - Provider：审批 / 驳回（带备注输入）
  - Aggregator：创建任务入口（内联展开任务创建表单）
- 目录文件预览：Aggregator 在需求面板内可查看目录文件，选中文件后直接创建任务

### 7.3 处理页

**列表区**：

- 搜索 + 状态筛选
- 卡片：任务类型 + 状态 + 进度条 + 处理器名称（或"手动"）

**详情面板**：

- 头部：任务 ID + 类型 + 状态徽章
- 任务配置信息（模型、模板等）
- 输入文件列表
- 实时进度区：进度条 + 处理器上报的最新 message
- 产物区：完成后列出生成的文件，可下载
- 手动模式操作区：开始/完成/失败按钮 + 产物登记表单

**新建任务**：

- 既可从需求详情面板发起（预填需求和目录文件）
- 也可在处理页创建：选需求→选文件→选任务类型→配置→提交
- 全部使用内联表单，不弹窗

### 7.4 交付页

- 全宽列表展示，不需要双栏
- 搜索 + 时间范围筛选
- 每行展示：需求标题、文件名、样本数、交付时间、下载按钮

## 8. 管理页面设计

### 8.1 注册审核页

- 双栏模式
- 列表区按状态分组（待审核 / 已通过 / 已驳回）
- 详情面板：申请信息 + 角色分配下拉 + 审批/驳回按钮

### 8.2 用户管理页

- 双栏模式
- 列表区显示用户列表
- 详情面板：用户信息 + 启用/停用操作

### 8.3 审计日志页

- 全宽时间线视图（不使用双栏）
- 按日期分组
- 顶部筛选：操作类型下拉 + 时间范围选择器

### 8.4 处理器管理页

- 双栏模式
- 列表区显示处理器卡片（名称、状态指示灯）
- 详情面板：处理器信息、端点地址、心跳时间、关联任务统计

## 9. 通用组件

### 9.1 新增组件

| 组件 | 职责 |
|---|---|
| `MasterDetail` | 列表+详情面板双栏容器，管理选中状态和面板切换 |
| `ItemCard` | 通用列表项卡片，插槽化内容，支持选中态高亮 |
| `DetailPanel` | 右侧详情面板容器，提供标题区、内容区、操作区插槽 |
| `InlineForm` | 面板内嵌的创建/编辑表单容器，管理提交和取消 |
| `StatusBadge` | 紧凑状态徽章，颜色自适应 |
| `EmptyState` | 空状态提示，含图标和描述文字 |
| `Timeline` | 竖向时间线，用于仪表盘活动流和审计日志 |
| `ProgressBar` | 自定义进度条，支持颜色和标签 |
| `FileList` | 文件列表，集成预览/上传/删除操作 |
| `KanbanBoard` | 看板容器，管理多列卡片布局 |
| `KanbanColumn` | 看板单列，含标题和卡片列表 |
| `KanbanCard` | 看板卡片，展示摘要信息 |

### 9.2 现有组件处理

| 现有组件 | 处理 |
|---|---|
| `StatusPill` | 重写为 `StatusBadge`，更紧凑 |
| `OverviewMetric` | 删除，被看板替代 |
| `SharedFilePreviewPanel` | 集成到 `FileList` / `DetailPanel` |
| `CatalogAssetPreviewPanel` | 合并到 `FileList` |

## 11. 响应式策略

| 断点 | 行为 |
|---|---|
| > 1200px | 双栏正常显示 |
| 768px - 1200px | 双栏收窄，列表区减小到 35% |
| < 768px | 单栏，点击列表项全屏展开详情面板，提供返回按钮 |

看板在 < 768px 时退化为垂直堆叠。
