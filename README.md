# 数据流转与指令生成系统

当前仓库已补充一份从零启动的技术选型与部署方案，见：

- `docs/系统选型与部署方案.md`
- `docs/扩展模块与部署策略.md`
- `docs/异构项目接入规范.md`
- `apps/web`

这份方案覆盖：

- 当前两项核心功能的架构设计
- 适合你们团队的技术选型
- 异构个人项目的聚合接入方式
- 后续“拆书”“修改病句”等新功能的接入方式
- 面向私有部署和实验室环境的部署建议
- 平台统一 `FastAPI` 与 `Taskiq` 的调整思路
- 单库优先、文件直接入 `PostgreSQL` 的简化方案
- 一个基于 `Vue 3 + TypeScript + Vite + Element Plus` 的前端骨架
